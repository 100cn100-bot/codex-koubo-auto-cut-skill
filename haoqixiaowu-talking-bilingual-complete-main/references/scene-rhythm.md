# Scene Rhythm

## Goal

Preserve the original spoken progression of the script.

Treat the script as performance rhythm:

- pauses
- repetitions
- reversals
- objections
- examples
- CTA cadence

## Default Splitting Method

Split in original order using these common beat types:

1. Hook
2. Objection
3. Response
4. Analogy
5. Explanation
6. Example
7. Workflow
8. Outcome
9. CTA

## Keep These

- repeated questions when they add emphasis
- repeated claims when they create rhythm
- transition phrases such as “但是”“你有没有想过”“比如说”“那当然”
- case-study pivots

## Compress Only When

- two adjacent lines do the same job
- repetition adds no rhythm value
- the user explicitly asks for a shorter cut

## Recommended Duration

- 4-6s for hook beats
- 5-8s for simple explanation beats
- 8-12s for dense explanation or workflow beats
- 5-8s for CTA beats

## Final Check

Before finalizing:

- verify the scene order still sounds like the same speaker
- verify key examples were not collapsed away
- verify the CTA remains at the tail
