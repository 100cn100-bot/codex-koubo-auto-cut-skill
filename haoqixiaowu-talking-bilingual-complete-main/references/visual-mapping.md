# Visual Mapping

## Purpose

Convert semantic beats into visual intent while preserving both rhythm and bilingual readability.

## Mapping Table

### Numbers, ratios, outputs, costs

Use:

- KPI scenes
- percentage scenes
- compare bars
- metric cards
- metric flow scenes

### Company names, platforms, products

Use:

- icon relationship scenes
- platform comparison scenes
- brand-labeled cards

### Complex concepts

Use:

- agenda lists
- explanation cards
- workflow loops
- document-style annotated scenes

### Workflow scenes

Use:

- process flows
- staged step scenes
- input -> output scenes

### Objections and doubts

Use:

- dialogue bubbles
- warning scenes
- contrast scenes

## Density Rule

Not every scene needs the same amount of packaging.

- simple hook: one strong bilingual title scene
- dense idea: one explanation scene
- workflow: one process scene
- data claim: one metric scene

## Bilingual Constraint

When choosing components, prefer ones that can hold:

- Chinese main text
- English support text

without forcing overflow.

If a component is too tight for bilingual use, simplify the English or use a more spacious component.
